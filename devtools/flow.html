📦 Release Workflow: full-release.sh

Navigation: 📦 Commit 🛡️ Checks 🔢 Version 📚 Changelog 📌 Tag 🌐 Push 🎉 Done
📦 1. Commit-creation
  • Commit type selection
  • Scope & description input
  • git commit is generated
  • release.log is supplemented
🛡️ 2. Security checks
  • Checks for a clean Git working directory
  • Warning for sensitive files
  • Check: New commits since last Git tag?
🔢 3. increase Version
  • Version read from version.txt
  • choose Patch / Minor / Major
  • write new version to version.txt
📚 4. create Changelog
  • Commits grouped by type
  • Entry in changelog.md
📌 5. Release-Commit & Tag
  • Commit of version & changelog
  • git tag is set
🌐 6. Optionaler Push
  • Push to origin HEAD + tag
  • Log entry: „Pushed …“
🎉 7. Final
  • Message „Release completed“
  • Final log entry
